Voy a describir qué hace cada cosa de las dependencias, para ver si nos interesa o no:

-----------------LIBRERÍAS PARA CARGAR IMÁGENES -------------------

CEGUI_BUILD_CORONA: es una librería de entrada/salida que puede leer, escribir y manipular fichero de imagen en unas pocas líneas de código. Puede escribir en fichero PNG y TGA y leer PNG, JPGE, PCX, ... etc Básicamente sirve para añadir carga de imágenes.

			http://corona.sourceforge.net/


CEGUI_BUILD_DEVIL: es una librería para desarrollar aplicaciones con mucho poder en la carga de imágenes. Igual que corona, carga, guarda, convierte, manipula, filtra y monitoriza distintos formatos. Soporta OpenGL, SDL, DirectX

			http://openil.sourceforge.net/


CEGUI_BUILD_FREEIMAGE: es una librería para soportar distintos formatos (Igual que corona y devil)

			http://freeimage.sourceforge.net/

CEGUI_BUILD_SILLY: es una librería que proporciona la carga de imágenes en el contexto de CEGUI. La idea es que reemplace DevIl con menos formatos de imagen pero centrado sobre todo en la carga de imágenes. (?)

			http://cegui.org.uk/wiki/SILLY


/-----------------LIBRERÍAS PARA CARGAR IMÁGENES -------------------/

CEGUI_BUILD_TINYXML: es una librería para parsear XML y crear objetos que reprensentan el XML.

			https://sourceforge.net/projects/tinyxml/

CEGUI_BUILD_XERCES : otro parseador más

			http://xerces.apache.org/xerces-c/

--------------- LIBRERÍAS DE SAMPLES(?) (OPENGL) ------------------

CEGUI_BUILD_GLEW: es una librería (extensión) que proporciona mecanismos de ejecución eficientes para determinar qué versiones de OpenGL son soportadas en la plataforma objetivo.

			http://glew.sourceforge.net/

CEGUI_BUILD_GLFW: es una librería para el desarrollo en OpenGL. Proporciona una API para crear ventanas, contextos y superficies, recibiendo input y eventos.

			http://www.glfw.org/

/--------------- LIBRERÍAS DE SAMPLES(?) (OPENGL) ------------------/




---------------- LIBRERÍAS DE RENDERIZADO --------------------------

CEGUI_BUILD-GLM: es una librería de matemáticas para gráficos basados en especificaciones GLSL. Proporcioa clases y funciones diseñadas e implementadas con la misma nomenclatura y funcionalidad que GLSL, así que si se sabe de GLSL se puede usar GLM en C++. Proporciona además: transformaciones de matrices, quaterniones, empaquetado de dados, números random, ruido, etc. También viene bien para raytracing y rasterización, procesado de imágenes, física.......WOW

			https://glm.g-truc.net/0.9.8/index.html

/---------------- LIBRERÍAS DE RENDERIZADO -----------------------/
